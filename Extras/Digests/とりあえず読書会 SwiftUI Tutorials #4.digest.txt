■ ダイジェスト

技術書を読み進める様子を録画した回です。SwiftUI Tutorials のチャプター 1「Building Lists and Navigation」のセクション 2〜6 を中心に、読み進めながら次を実装・確認しました。行ビュー `LandmarkRow` の作成（`var landmark: Landmark` を追加し、`HStack` に `Text(landmark.name)`、画像、`Spacer()` を配置）。独自画像型から SwiftUI の `Image` への変換イニシャライザを `extension SwiftUI.Image { init(_ image: Landmark.Image) { self.init(image.name) } }` として用意し、Row 側では `Image(landmark.image)` と書けるように整理。プレビューの設定では `#Preview` を用いて、`Bundle.main` の `landmarkData.json` から読み込んだデータを渡す形にし、複数プレビューと `Group`、および `.previewDisplayName("…")` を使ってケースを切り替え。プレビュー専用データ `landmarksForPreview` を `private` に分離し、リリースを汚さないよう `#if DEBUG … #endif` で囲う方針を確認。リスト作成では `LandmarkList` を作り、最初はスタティックに `LandmarkRow` を並べてから、`List(landmarks, id: \.id) { landmark in LandmarkRow(landmark: landmark) }` へとダイナミック化。続けて型 `Landmark` を `Identifiable` に準拠させ、`id` 指定を省略可能に。あわせて `Identifiable`（同一性）と `Equatable`（同値性）、`Hashable` の違いと注意点を整理し、`==` を `id` のみで比較する独自実装は避けるべきという指針を共有。ナビゲーションの導入では `NavigationSplitView` に `List` を埋め込み、行を `NavigationLink` で包んで遷移先を `LandmarkDetail(landmark: landmark)` に設定し、`navigationTitle("Landmarks")` を付与。トレーリングクロージャ構文による `NavigationLink { … } label: { … }` の読み方も押さえました。レイアウト調整では、テキストの左寄せが効かない問題を `VStack(alignment: .leading)` を適切なスタックに付け直すことで解決。プレビューではバインディングが必要な箇所に `.constant(...)` を用いて動作を確認しました。全体として、チュートリアルの指示に従い手を動かしつつ、設計上の選択（変換イニシャライザ、`Identifiable` の導入、プレビューの分離など）に着目して学習を進めています。
